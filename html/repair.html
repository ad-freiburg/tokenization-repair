Help

Select a correction method and click the button to submit your query. The corrected sequence will be shown below the input form.

Hyperparameters for this demo are set such that the models work best with unreliable spaces (e.g. no spaces at all). Query times are longer than reported in the paper, because the demo is currently set up on a CPU instead of GPU.

Approaches

Name Description
Bidirectional labeling model Bidirectional LSTM predicting space probabilities.
Beam search forward Beam search with a forward language model.
Beam search backward Beam search with a backward language model.
Two-pass beam search First the forward beam search, then the backward beam search on the result.
Beam search bidirectional Beam search using the forward language model in combination with the bidirectional model. This method gave the best results.
Spelling correction Forward beam search doing all kinds of character edits, not just space edits. Beware of longer query times due to the increased beam width.

Mode Description
Non-robust The models are trained on clean text.
Robust The models are trained on text with spelling errors. This setting gives much better results on sequences containing spelling errors.

For a comparison with the Google spell checker, copy your query into a Google document and iteratively apply all suggested space edits.
